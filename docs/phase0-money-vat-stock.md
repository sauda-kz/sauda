# SAUDA-067 — Phase 0: деньги, НДС, наличие

Техническое решение для единообразного хранения денежных значений, признака НДС и остатков до задач импорта прайсов и матчинга.

## Деньги

- **Тип в БД:** `NUMERIC(18,2)` — точное десятичное значение, не `float` / `double`.
- **Тип в Java:** `java.math.BigDecimal` с `@Column(precision = 18, scale = 2)`.
- Дробные цены поддерживаются; при хранении **не округлять** без явного бизнес-правила.
- В API значения передаются как точный `BigDecimal` в JSON (форматирование «999,90 ₸» — на фронте).

### Денежные поля (Phase 0)

| Сущность | Поле |
|----------|------|
| `offer` | `price` |
| `lot` | `budget_amount` |
| `lot_match` | `estimated_unit_price`, `estimated_total_price`, `budget_amount`, `estimated_margin` |

Новые денежные поля должны следовать тому же правилу.

Округление допустимо только для ориентировочных показателей (например примерной маржи в UI) и должно быть явно обозначено как примерное.

## НДС

Поле `offer.price_includes_vat` — `BOOLEAN NULL`:

| Значение | Смысл | Отображение в UI |
|----------|-------|------------------|
| `true` | Цена включает НДС | «с НДС» |
| `false` | Цена без НДС | «без НДС» |
| `null` | Не указано / неизвестно | «НДС не указан» |

Sauda **не рассчитывает** НДС в MVP — только сохраняет признак из источника.

Миграция: `V5__phase0_money_vat_stock.sql` заменила enum `vat_status` на `price_includes_vat`.

## Остатки и наличие

На `offer` два независимых поля:

| Поле | Тип | Описание |
|------|-----|----------|
| `stock_quantity` | `INTEGER NULL` | Точное количество, если известно |
| `stock_status` | enum | Статус наличия |

### Значения `stock_status`

| Статус | Значение |
|--------|----------|
| `in_stock` | Есть в наличии |
| `low_stock` | Мало / ограниченное наличие |
| `out_of_stock` | Нет в наличии |
| `on_order` | Под заказ (не подтверждён складской остаток) |
| `unknown` | Неизвестно |

### Маппинг текстовых остатков

Реализован в `com.sauda.domain.stock.StockTextMapper` (для будущих адаптеров импорта):

| Текст из прайса | Статус |
|-----------------|--------|
| много, в наличии, есть | `in_stock` |
| мало, ограничено, остаток малый | `low_stock` |
| нет, 0, отсутствует | `out_of_stock` |
| под заказ, ожидается, по запросу | `on_order` |
| пусто / не распознано | `unknown` |

Число `> 0` → `stock_quantity = N`, `stock_status = in_stock`.  
Число `0` → `stock_quantity = 0`, `stock_status = out_of_stock`.

## Правило `on_order` в матчинге

`on_order` **не считается** полноценным наличием (`LotMatchCalculator`):

- `stock_check` = `unknown`
- `quantity_check` = `unknown` (количество не доверяется)
- `needs_manual_review` = `true`
- В `risk_flags` добавляется `on_order_delivery_unconfirmed`

UI: «под заказ / нужно проверить срок поставки».
